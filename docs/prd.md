# AI Counsellor - Hackathon 需求文档

## 1. 应用概述

### 1.1 应用名称
AI Counsellor

### 1.2 应用描述
AI Counsellor 是一个基于阶段引导的平台，旨在帮助学生做出自信且明智的留学决策。该平台通过结构化的 AI 顾问系统，深入了解学生的学术背景、目标、预算和准备情况，并逐步引导他们从个人资料建立到大学筛选和申请准备。AI 顾问不仅回答问题，还主动推理、推荐、解释风险、筛选大学、锁定决策，并根据用户当前阶段创建可执行任务。这不是一个聊天机器人或浏览平台，而是一个决策和执行系统，旨在消除困惑，提供清晰度、方向和动力。

## 2. 核心功能\n
### 2.1 用户流程
完整的用户流程包括以下步骤：
- Landing Page（着陆页）\n- Signup / Login（注册/登录）
- Mandatory Onboarding（强制性入职引导）
- Dashboard with stage indicators（带阶段指示器的仪表板）
- AI Counsellor interaction（AI 顾问交互）
- University discovery and shortlisting（大学发现和候选名单）
- University locking（大学锁定/承诺阶段）
- Application guidance with actionable to-dos（带可执行待办事项的申请指导）

每个步骤必须逻辑性地解锁下一个步骤。

### 2.2 结构化入职引导
入职引导体验需要：
- 收集学术背景、学习目标、预算和考试准备情况
- 可通过手动填写或 AI 引导交互完成（语音可选）
- 在完成入职引导之前阻止访问 AI 顾问

收集的数据必须为所有未来推荐提供支持。

### 2.3 AI 顾问（核心功能）
AI 顾问需要实现以下功能：
- 理解用户的个人资料和当前阶段\n- 解释个人资料的优势和差距
- 推荐大学（Dream / Target / Safe 三个层级）
- 清楚解释为什么某所大学适合或存在风险
- 能够执行操作，包括：
  - 将大学加入候选名单
  - 锁定大学
  - 创建和更新待办任务

AI 顾问应引导决策，而不仅仅是回答问题。
\n### 2.4 大学发现与逻辑
大学发现系统需要：
- 使用研究过的、免费的或虚拟的大学数据
- 根据以下条件筛选大学：
  - 用户个人资料
  - 预算\n  - 国家偏好
- 显示费用、风险级别和录取可能性（简化逻辑即可）

不要求完美，逻辑推理是关键。
\n### 2.5 大学锁定机制
引入明确的承诺步骤：\n- 要求用户至少锁定一所大学
- 在锁定大学之前阻止申请指导
- 允许稍后解锁，但需明确警告

此步骤对于展示专注和决策纪律至关重要。\n
### 2.6 申请指导与待办事项
在锁定大学后：
- 显示所需文件和时间表
- 生成 AI 驱动的待办事项（SOP、考试、表格等）
- 允许将任务标记为完成并自动更新

不需要真实提交，指导和结构是目标。

## 3. 技术要求

### 3.1 前端\n- 使用 React 或 Next.js\n- 响应式设计，适配不同屏幕尺寸
\n### 3.2 后端
- 使用 FastAPI（Python）或 Node.js\n\n### 3.3 数据库
- 使用 PostgreSQL\n- 必须可扩展、可靠且结构良好

### 3.4 API
- 对大学 API 或任何可用于此任务的免费 API 进行相关研究

### 3.5 AI 模型
- 团队可以使用 Gemini 或任何其他合适的 AI 模型来支持 AI 顾问和相关逻辑

## 4. 其他要求

### 4.1 开发原则
- 优先考虑端到端的工作流程，而非高级功能
- 避免使用逻辑破损的模板
- 专注于清晰度、可用性和决策指导

### 4.2 项目目标
构建一个功能性原型，展示学生如何从困惑逐步走向清晰的留学之旅。不需要构建完整或生产就绪的系统，而是构建一个可工作的、逻辑性的、现实的功能原型。
